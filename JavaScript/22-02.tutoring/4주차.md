### 1. 웹 데이터처리 기술   

1. gzip: GNU zip
  - 사용이유: 사이트 트래픽의 대부분을 점유하는 big3요소(js, html, css) 를 90%가까이 압축 가능
2. 스테가노그라피
  - 전달하려는 기밀정보를 이미지 또는 비디오 안에 <b>숨기는</b> 심층 암호 기술
  - FF D8: 파일 시작
  - FF D9: 파일의 끝
  - 파일의 끝(FF D9)에 쓰는 것 16진수 변환했을 때 알 수 있음

3. br
4. SSO(Single Sine On)
  - 한번의 인증과정으로 로그인 하면 유지되는 것
  - accessToken, OAuth
5. 데이터 처리 규격 (서버랑 웹이 통신하는 규격)
  - JSON
    - 주의
      - UTF-8만 허용
      - 주석 작성 불가
      - 관계형db사용시 사용할 수 있으나, 관계형db의 장점인 인덱싱 활용 불가
    - 단점
      - 불필요 트래픽 버퍼: 파일이 텍스터 기반이라 무거움
      - 까다로운 뭄법
      - 확장 힘듦
      - 쌍따옴표 강제
      - 무거운순서: xml, json, yaml
  - YAML
    - 특징
      - utf-8, 16지원
      - 주석지원
      - 앵커, 별칭 지원
  - XML
    - 장점
      - 태그로 파일을 넘겨주고 받는 것 자체가 장점 (태그로 의미 유추 가능)
